2.1 梯度和梯度下降

2.1.1 代价和损失

刚开始训练时,我们会随机赋予权重矩阵随机的值,随后输入图像,很显然预测结果会是混乱的这时候要给计算机一个评定结果是好是坏的标准,这就是训练单个样本的代价**(差的平方和)也就是损失**

代价的平均值叫经验风险,代价函数也是一种函数

2.1.2如何优化

由于输入值太多,不可能微积分求解,所以随意选值,决定函数向左还是右优化以得到代价函数的MIN

==显然这样求出的是局部最小值==

现在升维到二维(高维):

对于多元微分,上升最快的方向是梯度的方向,那么下降最快的方向就是梯度的反方向,将输入的权重看作向量,那么梯度也是向量

对于代价函数,取得结果为总体平均值,这样对整体识别能力更好

这就是梯度下降,btw,我们希望代价函数是平滑的,便于找到MIN

2.1.3 另一种理解

梯度向量得出的结果实际上也是一种权重,告诉我们改变哪个参数性价比最高,参数越大,方向越陡峭,改变性价比越高

2.1.4 梯度消失

  • 直觉叙述: 我们知道深度网络需要反向传播, 如果信号在传递过程中越来越小, 传到前面的层时几乎为0, 前面的层就几乎学不到东西, 这就是宏观上的梯度消失

  • 数学叙述: [反向传播的本质是很多导数相乘](#3.1.3 反向传播的本质)以L层网络为例子

其中$J^{(L)}$包含本层激活函数的导数和权重的雅可比, 若这些矩阵点谱范数多数小于1, 就会被连乘压扁

比如:

  • 激活饱和: sigmoid 的导数小于等于0.25 , tanh的导数小于等于1, 且在两端接近0, 多层相乘衰减快
  • 深度高: CNN/MLP或者是回传步数多
  • 初始化: 权重矩阵特征值过小或者过大
2.1.5 学习率

宏观：学习率是一种衡量每次进行学习时，学习的内容的多少的超参数
想象一下下山的情景。要想安全到达山底，速度必须足够快，才能取得有意义的进展，但又不能太快，以免失去控制，跌跌撞撞。最佳学习率设定了安全的下山速度

机器学习：学习率是一个超参数(==只是一个控制更新权重的常数==)，用于控制机器学习模型在优化算法的每一步中调整参数的程度

为什么学习率很重要？

  • 在最佳学习率下，外星人会得出结论，鸡也是动物。如果是这样的话，这一定意味着腿的数量不是决定某物是否是动物的关键因素。

  • 如果外星人的学习率低，就无法从它这只鸡身上获得足够的洞察分析。外星人会得出结论，鸡不是动物，因为它们没有四条腿。外星人的学习能力很弱，在看到更多的鸡之前，它无法更新自己的思维。

  • 在高学习率下，外星人会过度纠正。现在，它会得出这样的结论：因为鸡是一种动物，而且鸡有两条腿，所以所有动物都必须有两条腿。高学习率意味着模型一次学习的东西“太多”

3.1 反向传播算法

3.1.1 训练过程

在前面已经得知:对于正权重的神经元,需要调大其激活值;对于负权重的神经元,需要减少其激活值;这样使得总输出减小

那么对于一个随机的结果,我们需要增大其标记参数的输出值,减小其它非标记参数的输出值,如果我们根据它需要更改的程度来记录每个标记或者非标记参数的激活值需要增大或者减小的程度,最后将这些程度加和起来,得到一个总的改变趋势,也是总改变权重(向量)

这就是反向传播,传播出了对上一层如何改变神经元激活值的指示

不断对上一层进行反向传播,得到初始的期望,从而改变权重,这就是反向传播算法

刚才我们关注的是单个样本对权重偏置的影响,如果只按照此权重更改,会将所有图像识别为一种图像,若想识别1-9,应该将训练集每一个样本过一次反向传播,为了整体识别效果最佳,取1-9的向量的平均值

==此处得到的平均值,不严格地说就是上一节的负梯度==

注:像素组成的图像有无数种形式,1-9只是其中的几种,因平均向量会让整体识别的更好

3.1.2 训练简化

如果训练集每一个样本都要过一次反向传播那就太费时间了,所以我们将训练集随机划分为许多minibatch,每个minibatch得出的结果当然不是最好的结果,但是是一个非常好的近似,这减少了我们的计算量

这个技巧叫做随机梯度下降

4.1 k折交叉验证

此时已经得到了模型和数据集，需要判断模型是否拟合，可以将数据拆分为训练集和测试集，如果只评估一次结果，很可能不准确，预期多评估模型，来确认可靠

此时有一个k的参数，引用给定数据样本要拆分为的组数，因此叫k折交叉验证，使用有限的样本来估计模型在用于预测模型训练期间未使用的数据时通常预期性能

4.1.1一般程序

一般程序如下：

  1. 随机排列数据集。
  2. 将数据集拆分为 k 组
  3. 对于每个唯一组：
    1. 将组作为维持数据集或检验数据集
    2. 将剩余组作为训练数据集
    3. 在训练集上拟合模型并在测试集上评估模型
    4. 保留评估分数并丢弃模型
  4. 使用模型评估分数样本总结模型的技能

这种方法涉及将观测值集随机划分为大小大致相等的 k 组或折叠。第一个折叠被视为验证集，该方法适用于剩余的 k − 1 个折叠。[1]

在使用**交叉验证（cross-validation, CV）**来评估机器学习模型的表现时，有一个非常重要的原则：所有数据准备（如标准化、编码、特征选择等）以及超参数调优，都必须只在每次交叉验证划分出来的训练集上进行，而不能在整个数据集上提前做

4.1.2 k的配置

为k选择值的常见策略如下三种:

1.具有代表性的: k值划分为的k个测试和数据集足够大以至于每个集都能代表整个数据集的特征

2.k = 10: 固定为10,这是一个有惊讶的折数,有低偏差和适度方差

3.k = n,逐个留出法,每一折只留一个样本进行测试Leave-One-Out Cross-Validation（LOOCV）

数据少时效果好，但计算代价高、波动大

创建简单网络

训练过程:

  1. 前向传播（Forward Propagation）： 在前向传播阶段，输入数据通过网络层传递，每层应用权重和激活函数，直到产生输出
  2. 计算损失（Calculate Loss）： 根据网络的输出和真实标签，计算损失函数的值
  3. 反向传播（Backpropagation）： 反向传播利用自动求导技术计算损失函数关于每个参数的梯度
  4. 参数更新（Parameter Update）： 使用优化器根据梯度更新网络的权重和偏置
  5. 迭代（Iteration）： 重复上述过程，直到模型在训练数据上的性能达到满意的水平

7. 清华科协 –神经网络和基础(总结)

7.1.1 神经网络的组成

输入(input): 输入信号的权重, 可以是外部输入或者是来自于其它神经元的输出

权重(weights): 输入信号的权重, 表示每个输入对神经元影响的大小

偏置(bias): 每个神经元的偏移量, 即为wx+b中的b, 保持输入的灵活性, 控制输入数据的阈值

激活函数(Activation): 对加权输入的结果应用非线性变换, 常见的有ReLU, Sigmoid, Tanh等

6.1.2 抽象表达
  • 是一个具有数以万计的特定参数的高维映射, 对于待定参数可以求偏导(权重矩阵和偏置), 具有强大的表示能力
  • 形式上来说$f_{\theta}(x) = y$, x就是向量输入, y是向量输出
  • 实际上这个定义非常的宽泛, 所以神经网络可以做很多事情
  • f(图片) = Possibility(用于图像分类)
  • f(上文) = next token(LLM)
  • f(x) = v(文字生成图片)
6.1.3 两个问题
  • 待定参数的形式(“神经网络”)是对的吗? 这样设计的函数的表示能力有上界吗? 会不会这么设计的函数, 根本找不到一组令人满意的参数来组成这样的函数? (任意一种形状的复杂函数)
    1. 此处不要求闭式解
    2. 泰勒展开, $f(x) = [1, x, x^2 …]dot[w_0, w_1, w_2…]$
    3. 傅立叶展开, $f(x) = [1, sin(x), sin(2x)…cos(x), cos(2x)…]dot[c, a_1, a_2…b_1, b_2…]$
    4. 上述都可以无限逼近(前者描述充分好的光滑函数, 不断提高n, 后者是周期函数)
    5. 逼近定理: 对于==有限定义域上==的==连续函数==f, 如果网络具有一个隐藏层, 并且使用非线性激活函数(sigmoid, ReLU), 可以找到权重, 使得网络的输出以任意精度逼近f (只解决了存在性)
  • 假设说待定参数的形式是对的, 考虑到数以亿计的参数, 如何求解出来呢?
    1. 求解出来之后要有一个判据判定好坏, 可以理解为现有的函数到目标函数的距离, 也就是损失
    2. 用一个集合来近似所有区域的学习,即{(真实输入), (真实输出)}
    3. 梯度下降法来达到最优值, 需要思考修改量的大小和方向, 也要思考极小和最小的问题
    4. 如何得到梯度? –>[反向传播算法](# 3.1.4 求梯度和误差项)

可以自动计算梯度的工具

计算图是由一系列节点和边组成的有向无环图, 每个节点代表一个操作和变量, 边表示变量之间的依赖关系, 在深度学习中, 计算图记录了每个操作(+, *, dot …)如何组成最终的输出

在前向传播的过程中, 模型通过一系列的操作计算出最终的结果, 同时构建计算图

  • 当对张量进行操作时, 框架会创建一个新的节点, 记录操作和其输入输出
  • 输出成为下一个输入, 操作之间通过边连起来
  • 计算图的节点不仅记录操作, 还会记录输入输出张量以便计算梯度
7.1.2 存储空间(OOM警告)

OOM = out of memory

计算图需要占用内存空间来存储与向前传播过程中的每个操作相关的中间结果以及梯度. 在训练过程中, pytorch 会动态构建计算图, 每个操作会生成一个对应的节点, 并在计算图中记录这些操作及其输入输出:

  • 操作: 加法, 乘法, 激活函数
  • 输入张量: 这些张量是计算图中某个节点的输入
  • 输出张量: 这是计算图中某个节点的输出

由于计算图的每个节点都可能有梯度信息, 因此计算图的存储空间会随着模型的复杂度和输入数据的大小而增加. 大模型和大数据集会使得计算图占用更多的内存空间

7.1.3 行成时间(动态构建)

计算图的行程时间是指在pytorch之中, 每当执行一个操作时, 计算图就会动态地构建, 只有当张量的require_grad设置为true时, pytorch会追踪该向量的运损并且记录操作, 构建出一个计算图, 每次向前传播的时候, pytorch都是根据当前的操作动态更新计算图, 以便于支持后续的梯度计算. 此过程按需进行, 并且能够根据不同的输入动态调整

7.1.4 销毁时间(反向传播后)

在完成反向传播并且计算出梯度时, 计算图会被销毁, 在pytorch中, 计算图在执行backward()时被使用, 并且在计算完梯度后通常会被销毁, 为了节省内存, pytorch会在完成反向传播后, 自动释放计算贴图所占用的内存

然而, 如果我们设置了backward(retain_graph = True), 那么计算图就不会被销毁, 这对于需要多次反向传播的情况非常有用
